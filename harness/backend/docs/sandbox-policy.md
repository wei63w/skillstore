# Harness 权限沙箱策略

## 操作类型

- `FILE`: 文件读写。
- `COMMAND`: 命令执行。
- `NETWORK`: 网络访问。
- `CREDENTIAL`: 凭据访问。
- `GIT`: Git 操作。
- `BUILD`: 构建操作。
- `DEPLOYMENT`: 部署操作。

## 决策类型

- `ALLOW`: 低风险受控操作，可自动执行。
- `DENY`: 越权或禁止操作，不得执行。
- `REQUIRE_APPROVAL`: 高风险操作，必须人工确认。

## 默认规则

- 包含 `..`、`.env`、系统目录的目标拒绝。
- 凭据访问必须审批。
- 部署、生产目标、端口操作必须审批。
- 普通项目内文件操作允许。

## 审计要求

每次决策必须记录 run id、操作类型、目标、风险等级、决策结果和原因。
