# Harness 部署执行与回滚手册

## 当前阶段

006 阶段只启用部署 dry-run 和 rollback plan 记录，不自动执行真实云部署。

## Docker 构建

Docker 构建阶段先生成 dry-run 命令和构建意图，后续接入真实 Docker build 时必须记录：

- 镜像名称。
- 构建输入。
- 构建日志。
- 镜像摘要。
- 失败原因。

## 部署审批

以下操作必须审批：

- 端口开放。
- 权限变更。
- 线上环境写入。
- 反向代理或证书变更。
- 数据库初始化或销毁。

## 回滚

部署计划必须包含上一稳定镜像或上一稳定版本的回滚策略。部署失败时优先回滚；无法自动回滚时生成明确人工操作建议。
